# Flamapp AI R&D Assignment 

## Problem Statement

The objective of this assignment is to estimate the unknown parameters of the given nonlinear parametric curve using only the observed `(x,y)` coordinates.

The governing equations are

x = t*.cos(theta) - exp(M*.|t|)*.sin(0.3.*.t)*.sin(theta) + X
y = 42 + t*.sin(theta) + exp(M*.|t|)*.sin(0.3*.t)*.cos(theta)

Unknown parameters:

- **θ** : Rotation angle
- **M** : Exponential growth/decay factor
- **X** : Horizontal translation

Known bounds:

- \(0deg < Theta< 50deg)
- \(-0.05 < M < 0.05\)
- \(0 < X < 100\)
- \(6 < t < 60\)

The assignment evaluates solutions using the **L1 distance** between the observed and reconstructed curves.

---

# Understanding the Problem

Although the governing equations are known, the parameters

- θ
- M
- X

are unknown.

Additionally, the dataset only contains observed `(x,y)` coordinates and does **not** provide the corresponding parameter `t` for each point.

Therefore, this is formulated as a **nonlinear parameter estimation problem**, where the objective is to recover the parameter values that best explain the observed curve.

# Approach 1 : Differential Evolution + KDTree

Since the parameter `t` is unknown, the complete curve is first generated by densely sampling

\[
t \in [6,60]
\]

using `numpy.linspace()`.

For every candidate parameter set `(θ,M,X)`:

1. Generate the predicted curve.
2. Build a KDTree from the predicted points.
3. Find the nearest predicted point for every observed point.
4. Compute the total L1 distance.
5. Differential Evolution searches for the parameter combination that minimizes this L1 error.

### Why Differential Evolution?

- Global optimization
- Derivative-free
- Works well with nonlinear objective functions
- Supports bounded parameter optimization

---

# Approach 2 : Analytical Recovery of t

Instead of approximating the correspondence using KDTree, the parameter `t` can be recovered analytically.

Starting from

\[
x=t\cos(\theta)-e^{M|t|}\sin(0.3t)\sin(\theta)+X
\]

\[
y=42+t\sin(\theta)+e^{M|t|}\sin(0.3t)\cos(\theta)
\]

Multiply the first equation by \(\cos(\theta)\)

\[
(x-X)\cos(\theta)
=
t\cos^2(\theta)
-
e^{M|t|}\sin(0.3t)\sin(\theta)\cos(\theta)
\]

Multiply the second equation by \(\sin(\theta)\)

\[
(y-42)\sin(\theta)
=
t\sin^2(\theta)
+
e^{M|t|}\sin(0.3t)\cos(\theta)\sin(\theta)
\]

Adding both equations,

the exponential terms cancel,

and using

\[
\cos^2(\theta)+\sin^2(\theta)=1
\]

we obtain

\[
\boxed{
t=(x-X)\cos(\theta)+(y-42)\sin(\theta)
}
\]

This allows the latent parameter `t` to be estimated directly from every observed point.

The recovered `t` is substituted back into the original equations to reconstruct the predicted curve, and Differential Evolution is again used to minimize the L1 error.

This approach removes the need for KDTree and nearest-neighbor matching.

---

# Approach 3 : Comparison of Local Optimizers

The analytical formulation from Approach 2 was retained while replacing the global optimizer with local optimization algorithms.

The following optimizers were evaluated:

- Powell
- Nelder-Mead

Each optimizer minimizes the same L1 objective function, allowing direct comparison with Differential Evolution in terms of

- Final L1 Error
- Runtime
- Convergence Behaviour

Every approach is evaluated using

---

# Observations

- Dense curve sampling combined with KDTree successfully estimates the unknown parameters even when `t` is unknown.
- Recovering `t` analytically simplifies the optimization problem and eliminates nearest-neighbor matching.
- The analytical formulation produced the best overall results.
- Differential Evolution consistently outperformed the local optimization algorithms, indicating that the optimization landscape contains multiple local minima where local methods may converge prematurely.

---

# Advantages & Limitations

| Approach | Advantages | Limitations |
|-----------|------------|------------|
| Differential Evolution + KDTree | Does not require knowledge of `t`, robust global optimization | Computationally expensive due to dense sampling and nearest-neighbor search |
| Analytical t + Differential Evolution | Faster, simpler objective, direct point correspondence, lowest L1 obtained | Requires mathematical derivation of `t` |
| Powell / Nelder-Mead | Simple derivative-free local optimization, fast convergence | Sensitive to initialization, easily trapped in local minima, higher L1 than Differential Evolution |
